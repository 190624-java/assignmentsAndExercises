Extreme Programming

Good programming practices, taken to the extreme

  1. Five values

    1. Communication

      Stresses the importance of communication, with face-to-face discussion, use of whiteboard, etc.

    2. Simplicity

      Doing the most simple thing that will work. Only address given requirements.

    3. Feeback

      Through constant feedback, teams can identify areas for improvement and revise their practices.

    4. Courage

      Taking needed action, even though it may not be the popular consensus in the team.

    5. Respect

      Mutual respect among teammates ensures effective work conditions.


  2. Twelve practices

    1. The planning game

    2. Small releases

    3. Metaphor

    4. Simple design

    5. Testing

    6. Refactoring

    7. Pair programming

    8. Collective ownership

    9. Continuous integration

    10. 40-hour week

    11. On-site customer

    12. Coding standard

Test-Driven Development

TDD Cycle

  1. Add a test

  2. Run all tests and see if any new test fails

  3. Write some code

  4. Run tests and refactor code

  5. Repeat
